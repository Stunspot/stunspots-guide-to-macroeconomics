# Knowledge Packs — Stunspot’s Guide to Macroeconomics

The canon is packaged in three forms so different AI/RAG systems can load it without destroying source traceability.

`docs/` is navigation and guidance. The knowledge files are in `knowledge-packs/`:

- [`knowledge-packs/by-report/`](https://github.com/Stunspot/stunspots-guide-to-macroeconomics/tree/main/knowledge-packs/by-report) — canonical individual source reports.
- [`knowledge-packs/compiled-packs/`](https://github.com/Stunspot/stunspots-guide-to-macroeconomics/tree/main/knowledge-packs/compiled-packs) — grouped upload packs.
- [`knowledge-packs/omnibus/`](https://github.com/Stunspot/stunspots-guide-to-macroeconomics/tree/main/knowledge-packs/omnibus) — whole-corpus bundle.

---

## Recommended Default

Use the **compiled packs** unless you have a specific reason not to.

They give you four files instead of thirteen source reports or one large omnibus. That usually works best for AI Projects, RAG systems, NotebookLM-style tools, and long-context workspaces with practical file-count limits.

## Which Format Should I Upload?

### Use compiled packs when...

You want the best default model-facing upload format. The four compiled packs preserve the canon’s intellectual structure:

1. **A–C** — foundations of monetary reality, liquidity, and economic causality.
2. **D–F** — monetary governance, institutional control, and policy transmission.
3. **G–I** — market structure, asset regimes, and capital coordination.
4. **J–M** — information systems, strategic signaling, narrative liquidity, diagnostics, and strategy.

This format is the right choice for most hosted AI project knowledge bases.

### Use source reports when...

You need tighter retrieval boundaries, clean report-level citations, selective uploads, or editorial access to individual source units. This is best for custom RAG systems, research workflows, or any use case where provenance matters more than upload simplicity.

### Use the omnibus when...

You want the entire canon in one file for archival use, local search, local model testing, or a long-context system that performs well with large single-file inputs. The omnibus is convenient, but it gives the retrieval system fewer natural boundaries.

## RAG Hygiene Notes

Avoid indexing multiple packaging layers into the same collection unless you intentionally deduplicate. If you index both source reports and compiled packs, the same passage can appear twice with different paths, confusing provenance and citation logic.

For clean retrieval, choose one layer per collection:

- **Source-report collection:** best provenance.
- **Compiled-pack collection:** best default usability.
- **Omnibus collection:** best archive simplicity.
